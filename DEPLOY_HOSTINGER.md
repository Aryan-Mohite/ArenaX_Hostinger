# ArenaX ‚Äî Hostinger Business Plan Deployment Guide

## Architecture Overview

```
Hostinger Node.js App
‚îÇ
‚îú‚îÄ‚îÄ server.js          ‚Üê entry point (Hostinger points here)
‚îú‚îÄ‚îÄ src/               ‚Üê Express API backend
‚îú‚îÄ‚îÄ frontend/          ‚Üê React source (Vite)
‚îÇ   ‚îî‚îÄ‚îÄ dist/          ‚Üê Built output served by Express (generated by build step)
‚îî‚îÄ‚îÄ package.json       ‚Üê single package.json; postinstall auto-builds frontend
```

The Express server serves **both** the API (`/api/*`) and the React frontend
(static files from `frontend/dist/`) from the same process and port.

## Step-by-Step Setup

### 1 ‚Äî Upload Files

Use Git or Hostinger File Manager to upload the project root.  
**Do NOT upload** `node_modules/` or `frontend/dist/` ‚Äî these are generated.

### 2 ‚Äî Set Environment Variables

In **hPanel ‚Üí Node.js ‚Üí Environment Variables**, add every key from `.env.example`:

| Key | Value |
|-----|-------|
| `NODE_ENV` | `production` |
| `PORT` | `5000` *(Hostinger may override this)* |
| `DB_HOST` | Your MySQL host (usually `localhost`) |
| `DB_PORT` | `3306` |
| `DB_USER` | MySQL username |
| `DB_PASSWORD` | MySQL password |
| `DB_NAME` | Database name (e.g. `arenax`) |
| `JWT_SECRET` | Long random string (32+ chars) |
| `JWT_EXPIRES_IN` | `7d` |
| `EMAIL_HOST` | Your SMTP host |
| `EMAIL_PORT` | `587` |
| `EMAIL_USER` | SMTP username |
| `EMAIL_PASS` | SMTP password |
| `EMAIL_FROM` | `ArenaX <no-reply@yourdomain.com>` |
| `APP_URL` | `https://yourdomain.com` |

### 3 ‚Äî Create the MySQL Database

In **hPanel ‚Üí Databases ‚Üí MySQL Databases**:
1. Create a new database (e.g. `arenax`)
2. Create a database user and assign full privileges
3. Import the schema from `database/schema.sql` via phpMyAdmin

### 4 ‚Äî Install Dependencies & Build

Connect via **SSH** (hPanel ‚Üí SSH Access) and run:

```bash
cd ~/domains/yourdomain.com/public_html   # or wherever you uploaded

npm install        # installs backend deps AND auto-builds React frontend
                   # (postinstall script runs: cd frontend && npm install && npm run build)
```

If `postinstall` doesn't trigger automatically on Hostinger, run manually:

```bash
npm run build:frontend
```

You should see `frontend/dist/` created with `index.html` and asset files.

### 5 ‚Äî Configure Node.js App in hPanel

Go to **hPanel ‚Üí Node.js**:

| Setting | Value |
|---------|-------|
| Node.js version | 18 or 20 (LTS) |
| Application root | `/path/to/your/upload` |
| Application URL | Your domain |
| Application startup file | `server.js` |

Click **Create** / **Save**, then click **Start** (or Restart if already running).

### 6 ‚Äî Verify

- Open `https://yourdomain.com` ‚Üí React app should load
- Open `https://yourdomain.com/api/health` ‚Üí should return `{"status":"ok"}`
- Open `https://yourdomain.com/login` ‚Üí React Router handles it correctly

## SEO: Prerendering with react-snap (automated via GitHub Actions)

ArenaX is a client-rendered React app. Per-page titles, meta descriptions, and
JSON-LD (via `<SEO>`/react-helmet-async) only exist after JavaScript runs ‚Äî
crawlers that don't wait for that see generic/empty content on every page
except the homepage. `react-snap` fixes this by rendering each route to real
static HTML at build time, using headless Chromium (Puppeteer).

Since deploy here is **GitHub push ‚Üí Hostinger auto-deploy** (no manual
upload step), prerendering runs in **GitHub Actions** instead of locally or
on Hostinger ‚Äî see `.github/workflows/prerender-deploy.yml`. Puppeteer needs
system libraries (libnss3, libatk, etc.) that Hostinger's Node build
environment doesn't reliably provide, but GitHub's `ubuntu-latest` runners
have full Chromium support out of the box.

### How it works

1. You push source changes to `main` as usual.
2. GitHub Actions builds the frontend, runs `npm run prerender`, and commits
   the resulting `frontend/dist/` (real static HTML per route, from
   `reactSnap.include` in `frontend/package.json`) back to `main`.
3. That commit triggers Hostinger's auto-deploy again. `scripts/build-frontend.js`
   detects the prerendered dist is already present and skips rebuilding it,
   so Hostinger just serves the prerendered files as-is.

No manual steps required after the one-time setup (enable "Read and write
permissions" for Actions under repo Settings ‚Üí Actions ‚Üí General ‚Äî see
`README_PRERENDER_FIX.md`).

### Verify it worked

Open a page's raw HTML source directly (View Page Source, not dev tools ‚Äî
dev tools shows the post-JS DOM either way) and confirm the real
title/description/JSON-LD are present without running any JS:

```bash
curl -s https://arenax.io/tournament | grep -o '<title>.*</title>'
```

### Keeping the route list current

`reactSnap.include` in `frontend/package.json` only lists static routes. When
you add a new blog post, add its `/blog/:slug` path there too (same manual
sync step as `BLOG_SLUGS` in `src/app.js`). Individual `/tournament/:id`
pages are intentionally left out of prerendering ‚Äî they're DB-driven and
already covered by the dynamic sitemap; the homepage/listing pages carry the
keyword-targeting weight.

## Local Development

```bash
# Terminal 1 ‚Äî backend
npm install
node --env-file=.env server.js          # or: npm run dev  (nodemon)

# Terminal 2 ‚Äî frontend (Vite dev server with proxy to :5000)
cd frontend
npm install
npm run dev                             # opens http://localhost:5173
```

The Vite dev server proxies `/api/*` to `http://localhost:5000` automatically.

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `Cannot find module` errors | Run `npm install` again from project root |
| Blank page / 404 on routes | `frontend/dist/` missing ‚Äî run `npm run build:frontend` |
| API returns 500 | Check Hostinger Node.js logs; verify all `.env` vars are set |
| DB connection refused | Confirm `DB_HOST=localhost` and MySQL user has correct privileges |
| `Missing required environment variables` | App will refuse to start ‚Äî all env vars in the table above are mandatory |
